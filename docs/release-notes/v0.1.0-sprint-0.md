# Release Notes — v0.1.0 (Sprint 0: Project Foundation)

Release date: 2026-07-12
Sprint: Sprint 0 — Project Foundation
Sprint summary: `docs/sprints/sprint-00-project-foundation-summary.md`

## Highlights

This is the platform's foundation release. There are no customer-facing
commerce features yet — this release establishes the monorepo,
containerized local environment, design system, API standards, and
authentication skeleton that every later sprint builds on.

## Added

- **Monorepo**: pnpm workspaces + Turborepo across `apps/*`,
  `packages/*`, and `playwright`, with shared strict TypeScript and
  ESLint configuration.
- **Storefront & Admin app shells**: two independent Next.js 15 App
  Router apps (`apps/storefront`, `apps/admin`) with header/footer/
  navigation shells, sharing the `@ecom/ui` design system.
- **Design system (`@ecom/ui`)**: design tokens, shared Tailwind
  preset, `Button` and `Card` primitives, and a Storybook instance with
  accessibility checks (`addon-a11y`) and a design-tokens reference page.
- **Authentication skeleton**: OTP request/verify endpoints, JWT access
  tokens, rotating refresh tokens (hashed at rest), `Public`/`Roles`
  decorators, and an RBAC guard, under `/api/v1/auth`.
- **API standards**: versioned routes, a standard success/error
  response envelope, request-ID correlation, global validation
  (`whitelist`, `forbidNonWhitelisted`), structured logging, and
  auto-generated OpenAPI/Swagger docs.
- **Health checks**: `/health`, `/health/live`, `/health/ready` (the
  latter verifying live Postgres and Redis connectivity).
- **Rate limiting**: global request throttling plus stricter limits on
  the unauthenticated OTP endpoints.
- **Local environment**: Docker Compose stack for PostgreSQL, Redis,
  Meilisearch, MinIO, Mailpit, and Nginx, plus a documented
  `.env.example`.
- **Testing**: 42 unit tests, 6 API integration/e2e tests, and 6
  Playwright end-to-end tests (including automated accessibility
  scans of both app shells) — all passing.
- **CI**: GitHub Actions pipeline running install → typecheck/lint →
  unit + API e2e tests (against real Postgres/Redis containers) →
  build → a dedicated Playwright e2e job → a dependency
  security-audit job.
- **Documentation**: a Sprint 0 security review
  (`docs/security/sprint-00-security-review.md`) and 8 Architecture
  Decision Records covering the backend architecture, both frontend
  frameworks, the database, cache, search, storage, and design-system
  choices (`docs/decisions/0001`–`0008`).

## Fixed

- Closed a rate-limiting gap on the public OTP request/verify endpoints
  that had no abuse protection beyond the existing 5-attempt verify
  cap.
- Fixed WCAG AA color-contrast failures on the primary/destructive
  button variants, the storefront announcement bar, and the admin
  sidebar section labels (all found via the Storybook a11y addon and
  Playwright + axe-core checks).

## Security

- Full findings and remediation plan: `docs/security/sprint-00-security-review.md`.
- `pnpm audit --audit-level=critical` now runs in CI on every change;
  no critical advisories are present as of this release.

## Known Issues

- Local Next.js builds on Windows machines with the repository stored
  in a OneDrive-synced folder can intermittently fail or stall when
  `apps/storefront` and `apps/admin` build in parallel via Turborepo
  (`EPERM` on `.next/trace`). Build each app individually as a
  workaround; CI (Linux runners) and Docker builds are unaffected.

## Upgrade / Setup Notes

This is the first release — no upgrade path applies. To run locally:

1. `docker compose -f infrastructure/docker/docker-compose.yml up -d`
2. `pnpm install`
3. `pnpm turbo run build`
4. Start `apps/api`, `apps/worker`, `apps/storefront`, `apps/admin` per
   the root README.
