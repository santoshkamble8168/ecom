# Deployment Checklist

## Purpose
Checklist for validating Deployment work before handoff.

## Responsibilities
- Provide repeatable review criteria.
- Expose missing requirements, tests, or docs.
- Support consistent AI and human review.

## Best Practices
- Use before review and again before completion.
- Adapt depth to risk and blast radius.
- Record unresolved items as explicit risks.

## Checklist
- [ ] Requirements and acceptance criteria are covered.
- [ ] Architecture and ownership are clear.
- [ ] Security and privacy are reviewed.
- [ ] Performance and accessibility are reviewed where applicable.
- [ ] Tests and documentation are complete.
- [ ] Rollback or recovery is documented when relevant.

## Examples
- Use this checklist as a PR description section or release gate.

## Common Mistakes
- Checking boxes without evidence.
- Ignoring edge cases and failure modes.
- Skipping docs because implementation seems obvious.
